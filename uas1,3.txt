1.	- Single responsibility principle adalah prinsip pengembangan perangkat lunak yang memastikan bahwa setiap class hanya memiliki satu tugas utama saja. Keuntungannya adalah code menjadi lebih rapi, performance menjadi lebih cepat, jumlah line pada method lebih sedikit sehinggan membuat code lebih enak dimaintain dan penggunaan resource memory lebih sedikit.
- Open/closed principle adalah prinsip yang digunakan pada sebuah cource code dimana kita bisa merubah dengan cara menambah entitas tanpa merubah source code yang sudah ditentukan sebelumnya sehingga tidak perlu lagi mengubah sumber source code yang sifatnya tertutup.
